# 写入文档内容

## 格式：写入内容到指定 MDX 文件

### 场景1：新增文件
```
使用提示词03：文件路径：【ulearning/getting-started】 语言：【zh-CN】 内容：【这里填入文档内容，支持 Markdown 和 MDX 语法】
```

**带翻译同步**：
```
使用提示词03：文件路径：【ulearning/getting-started】 语言：【zh-CN】 内容：【...】 翻译同步：【是】
```
- 翻译同步【是】：AI 自动将内容翻译为 `en-US`、`fr-FR`、`ar` 三个语言版本并写入（目标文件已存在但正文为空时也翻译写入）
- 翻译同步【否】：仅写入指定语言，不翻译、不检查其他语言

**指定 Title（可选）**：
```
使用提示词03：文件路径：【ulearning/getting-started】 语言：【zh-CN】 Title：【快速入门指南】 内容：【...】 翻译同步：【是】
```

---

### 场景2：修改已有文件（覆盖全部内容）
```
使用提示词03：文件路径：【ulearning/getting-started】 语言：【zh-CN】 内容：【更新后的全部内容...】 翻译同步：【是】
```
- 不指定插入位置时，用新内容**完全覆盖**原文件
- 翻译同步开启时，对其他语言版本**正文已有内容**的文件不覆盖，以保护用户手动编辑的内容（仅 `zh-CN` 覆盖写入）

---

### 场景3：修改已有文件（在指定位置插入内容）
```
使用提示词03：文件路径：【ulearning/getting-started】 语言：【zh-CN】 插入位置：【## 安装步骤之后】 内容：【### 常见问题\n\nQ: 如何重置密码？\nA: 联系管理员重置。】 翻译同步：【是】
```
- 插入位置格式：`【## 标题名称之后】` 或 `【### 标题名称之后】`
- 在原文件中找到匹配标题，在其**之后**插入新内容，其余内容保持不变
- 翻译同步开启时，其他语言版本**会尝试翻译插入**：
  - 找到对应标题 → **翻译插入**
  - 找不到对应标题 → **跳过**（提示用户确认等效标题）

---

### 其他语言同步结果说明

| 目标文件状态 | 覆盖模式（无插入位置） | 插入模式（有插入位置） |
|---|---|---|
| 文件不存在 | 翻译生成（空文件新建） | 翻译生成（空文件新建） |
| 正文为空（仅有 title） | 翻译生成（空文件新建） | 翻译生成（空文件新建） |
| 正文有内容，标题匹配 | 已存在，跳过未修改 | 翻译插入（标题匹配） |
| 正文有内容，标题不匹配 | 已存在，跳过未修改 | 已存在，跳过（插入标题未找到） |

---

### 通用说明

- 文件路径相对于 `pages/` 目录，不包含语言后缀，后缀由语言参数自动拼接
- 提示词03仅负责写入 MDX 文件内容，不涉及目录结构或 `_meta.{locale}.json` 的创建/更新
- **修改场景下**：若原文件已有 `title`，且用户未指定新 `Title`，保留原 `title` 不变

**ar 语言后缀**：`ar` 语言的 MDX 文件后缀为 `ar`（而非 `ar-AR`），如 `getting-started.ar.mdx`

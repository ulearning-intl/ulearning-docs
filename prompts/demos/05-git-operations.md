# Git 提交流操作

---

### 格式1：提交本地修改
```
使用提示词05：操作：【提交】 提交信息：【添加智能排课功能文档】
```

### 格式2：提交并推送
```
使用提示词05：操作：【推送】 提交信息：【更新数据分析页面】
```

### 格式3：拉取远程最新代码
```
使用提示词05：操作：【拉取】
```

### 格式4：查看当前状态
```
使用提示词05：操作：【状态】
```

### 格式5：查看提交历史
```
使用提示词05：操作：【日志】 行数：【10】
```

---

### 流程说明

**提交（commit）**：仅将修改保存到本地仓库，不会推送到远程

**推送（push）**：AI 会先执行拉取（pull），确认无冲突后再推送。若存在冲突，AI 会停止推送并告知你具体冲突文件列表及解决建议

**冲突处理**：当检测到冲突时，AI 会列出所有冲突文件，并给出手动解决步骤（使用 VSCode 打开文件、选择保留版本、手动标记冲突已解决）

---

### 注意事项
- 提交信息由你提供，建议简洁描述本次修改内容
- 推送前 AI 会自动拉取远程代码，这是为了减少冲突
- 若真的产生冲突，AI 不会强制覆盖，会告知你具体哪些文件有冲突，由你手动处理
- 涉及多人协作的分支操作（如 merge、rebase），AI 会先询问你确认

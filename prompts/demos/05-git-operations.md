# Git 提交流操作

---

### 正确操作流程

对于不懂 Git 的运营人员，建议按以下流程操作：

**步骤1：每天开始更新文档前，需要先更新项目**
```
使用提示词05：操作：【拉取】
```
把项目更新到最新版，防止修改后和远程别人的修改冲突。

**步骤2：查看当前状态（可选）**
```
使用提示词05：操作：【状态】
```
先确认有哪些文件被修改了。

**步骤3：提交到本地仓库**
```
使用提示词05：操作：【提交】 提交信息：【简洁描述本次修改】
```
提交只是把修改记录到本地，不会丢失你的工作。

**步骤4：推送到远程（当你准备好分享代码时）**
```
使用提示词05：操作：【推送】 提交信息：【同上一步的描述】
```
AI 会先检查本地状态，如果有未提交修改会先提交到本地，然后拉取远程最新代码，确认没有冲突后再推送。如果有冲突，会告诉你哪些文件有冲突以及如何解决。

---

### 操作格式示例

```

```
使用提示词05：操作：【状态】

---

### 流程说明

**提交（commit）**：仅将修改保存到本地仓库，不会推送到远程

**推送（push）**：AI 会先检查本地状态，如有未提交修改先提交到本地，然后执行拉取（pull），确认无冲突后再推送。若存在冲突，AI 会停止推送并告知你具体冲突文件列表及解决建议

**冲突处理**：当检测到冲突时，AI 会列出所有冲突文件，并给出手动解决步骤（使用 VSCode 打开文件、选择保留版本、手动标记冲突已解决）

---

### 注意事项
- 提交信息由你提供，建议简洁描述本次修改内容
- 推送前 AI 会自动拉取远程代码，这是为了减少冲突
- 若真的产生冲突，AI 不会强制覆盖，会告知你具体哪些文件有冲突，由你手动处理
- 涉及多人协作的分支操作（如 merge、rebase），AI 会先询问你确认

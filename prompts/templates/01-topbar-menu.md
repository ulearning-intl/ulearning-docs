# 提示词 01：生成顶部栏导航项

## AI 读取说明

本文件为「AI 自读模式」提示词。用户无需复制文件内容，只需在对话框中输入：

```
使用提示词01：【导航类型】【顶部栏名称】【子项】【位置】【侧边栏】
```

示例：
```
使用提示词01：导航类型：下拉菜单（新增） 顶部栏名称：高级产品 下拉子项：智能排课,数据分析(单页面) 插入位置：插在产品之后
```

---

## 执行逻辑

当用户输入「使用提示词01」开头的指令时，AI 需按以下步骤执行：

### 步骤1：解析用户参数

从用户输入中提取以下信息：
- **导航类型**：`下拉菜单（新增）`、`单页面（新增）`、`下拉菜单（修改）`、`单页面（修改）`（必填）
- **顶部栏名称**：如「高级产品」（必填）
- **下拉子项**：格式为 `名称(单页面)` 或 `名称`，多个子项用逗号分隔（仅下拉菜单需要）。标 `(单页面)` 则单页面，不标则默认有侧边栏
- **插入位置**：如「插在产品之后」「插在文档和解决方案之间」「保持原位」（必填）
- **侧边栏**：`单页面`（仅单页面需要，可选，不填则默认有侧边栏）

### 步骤2：项目约束

1. **多语言同步**：`en-US`、`zh-CN`、`fr-FR`、`ar` 四个语言目录必须同步新增文件（以 `next.config.js` 中 `i18n.locales` 为准）
2. **Frontmatter**：每个 MDX 文件头部必须有 `title` 字段，**正文内容为空**，不自动生成任何描述性文字
3. **目录结构参考**：
   - 有侧边栏：参考 `pages/ulearning/`（含 `_meta.{locale}.json`），子目录的 `_meta.{locale}.json` 格式为简单键值对：`{"slug": "标题"}`
   - 单页面：参考 `pages/blog.{locale}.mdx`（直接在 pages 目录下的 MDX 文件）
4. **导航配置**：**只有 `pages/_meta.{locale}.json` 根目录配置控制顶部栏显示**，子目录（如 `pages/ulearning/_meta.{locale}.json`）中的配置只影响侧边栏导航。在根 `_meta.{locale}.json` 中添加或修改条目，顺序即显示顺序
5. **隐藏条目规则**：当下拉菜单中的子项链接到一个独立的页面目录（如 `/test2/overview`），且该目录不是顶部栏的直接导航项时，必须在根 `_meta.{locale}.json` 中添加 `"display": "hidden"` 条目，否则该目录会自动出现在顶部栏中
6. **侧边栏规则**：侧边栏的有无由文件结构决定，与 `theme.sidebar` 无关
   - **有侧边栏**（默认）：创建文件夹结构，如 `pages/ulearning/`，包含 `_meta.{locale}.json` 和入口 MDX 文件
   - **单页面**：创建单个 MDX 文件，如 `pages/blog.{locale}.mdx`，直接在 pages 目录下

### 步骤3：生成内容

#### 3.1 链接自动生成规则（仅新增时使用）

根据名称自动生成路由路径，规则如下：
- 中文名称：翻译为英文，空格用连字符 `-` 连接，全部小写，如「智能排课」→ `smart-scheduling`
- 英文名称：全部小写，空格用连字符 `-` 连接，如「API Docs」→ `api-docs`
- 有侧边栏时（默认）：路径为 `/{kebab-case-name}/introduction`（入口页面）
- 单页面时：路径为 `/{kebab-case-name}`（直接在 pages 目录下的 MDX 文件）

#### 3.2 新增导航（下拉菜单/单页面）

根据导航类型和侧边栏设置生成：

##### 3.2.1 单页面（新增）

- **有侧边栏**（默认）：创建 `pages/{path}/` 目录，在目录中创建 `_meta.{locale}.json` 和入口 MDX 文件（如 `introduction.{locale}.mdx`），链接格式：`/{path}/introduction`
- **单页面**：创建 `pages/{path}.{locale}.mdx` 文件（直接在 pages 目录下），链接格式：`/{path}`

##### 3.2.2 下拉菜单（新增）

下拉菜单的每个子项默认有侧边栏，标 `(单页面)` 的子项单页面：
- **有侧边栏**（默认）：子项创建 `pages/{item-path}/` 目录，包含 `_meta.{locale}.json` 和入口 MDX 文件，链接格式：`/{item-path}/introduction`
- **单页面**：子项创建 `pages/{item-path}.{locale}.mdx` 文件，链接格式：`/{item-path}`

#### 3.3 修改已有导航（下拉菜单/单页面）

- **下拉菜单（修改）**：更新 `pages/_meta.{locale}.json` 中对应菜单的子项名称，**保留原有链接不变**；若新增子项，按链接自动生成规则为新子项生成链接
- **单页面（修改）**：更新 `pages/_meta.{locale}.json` 中对应条目的标题，**保留原有链接不变**

### 步骤4：输出要求

1. 列出所有需要新建的文件及其完整内容（含文件路径）
2. 列出所有需要修改的文件及其具体改动内容（含文件路径和改动位置）
3. 描述完成后在站点顶部栏的效果

## 参数解析规则

| 参数 | 格式 | 示例 |
|---|---|---|
| 导航类型 | 下拉菜单（新增）/ 单页面（新增）/ 下拉菜单（修改）/ 单页面（修改） | 下拉菜单（新增） |
| 顶部栏名称 | 任意文本 | 高级产品 |
| 下拉子项 | `名称(单页面)` 或 `名称`，逗号分隔 | 智能排课,数据分析(单页面) |

## 示例

### 示例1：下拉菜单（新增）

**输入：**
```
使用提示词01：导航类型：下拉菜单（新增） 顶部栏名称：高级产品 下拉子项：智能排课,数据分析(单页面) 插入位置：插在产品之后
```

**预期执行：**
- AI 根据名称自动生成链接：「智能排课」→ `/smart-scheduling/introduction`（默认有侧边栏），「数据分析」→ `/data-analysis`（单页面）
- 在 `pages/_meta.{locale}.json` 中，在「产品」之后添加「高级产品」下拉菜单条目
- 在 `pages/_meta.{locale}.json` 中，为 `smart-scheduling` 目录添加 `"display": "hidden"` 条目（因为它只作为下拉子项链接，不单独出现在顶部栏）
- 创建 `pages/smart-scheduling/` 目录及 `_meta.{locale}.json`、`introduction.{locale}.mdx`（MDX 文件仅包含 title，正文为空）
- 创建 `pages/data-analysis.{locale}.mdx` 文件（MDX 文件仅包含 title，正文为空）
- 同步生成 `en-US`、`zh-CN`、`fr-FR`、`ar` 四个语言版本

**效果：**
顶部栏显示顺序：`产品` → `高级产品` → `文档` → `解决方案` → `最佳实践` → `变更日志`，点击「智能排课」有侧边栏，点击「数据分析」单页面

---

- 创建 `pages/api-docs.{locale}.mdx` 文件（四个语言版本）

**效果：**
顶部栏显示顺序：`产品` → `文档` → `解决方案` → `API文档` → `最佳实践` → `变更日志`，页面单页面

```
使用提示词01：导航类型：下拉菜单（修改） 顶部栏名称：产品 下拉子项：学习管理系统,智慧教室,新增功能(单页面) 插入位置：保持原位
```

**预期执行：**
- 更新 `pages/_meta.{locale}.json` 中「产品」菜单的子项名称，**保留原有链接不变**（`/ulearning/getting-started`、`/uclass/start`）
- 为新增子项「新增功能」自动生成链接：`/new-features`（单页面），并创建 `pages/new-features.{locale}.mdx` 文件

**效果：**
顶部栏顺序不变，「产品」下拉菜单中子项名称更新，新增子项「新增功能」单页面
